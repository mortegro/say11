# say-e11 — TTS CLI replicating macOS `say`

**Date:** 2026-05-17
**Status:** Approved design

## Purpose

A `uv`-installable command-line tool that speaks text aloud using a cloud TTS
provider, replicating the core behavior of macOS `say`. It selects between
ElevenLabs and Deepgram automatically based on which API key is available.

## Command

Installed via `uv tool install`; the CLI entry point is `say`. Safe on Linux,
where no system `say` exists.

### Usage

```
say hello world              # text from positional args
echo "hello" | say           # text from stdin (when no args given)
say -f notes.txt             # text from a file
say -o out.wav "hello"       # speak AND write a WAV file
say --provider deepgram ...  # force a provider
say -v <voice> "hello"       # choose voice
say -r 200 "hello"           # speaking rate in words per minute
```

### Flags

| Flag | Meaning | Default |
|------|---------|---------|
| `-f`, `--file PATH` | Read text from a file | — |
| `-o`, `--output PATH` | Also write audio to a WAV file | — |
| `-v`, `--voice NAME` | Provider voice ID/name | provider default |
| `-r`, `--rate WPM` | Speaking rate in words per minute | 175 |
| `--provider {elevenlabs,deepgram}` | Force a provider | auto-detect |

### Text input precedence

1. Positional arguments (joined with spaces)
2. `-f/--file` if no positional args
3. stdin if no positional args and no `-f`
4. If none of the above yield text → error to stderr, exit 1

## Provider selection

1. Load environment: read `./.env` then `~/.env` via `python-dotenv`.
   Values already in the process environment win; among files, `./.env` wins
   over `~/.env`.
2. If `--provider` is given, use it. Error if its key is missing.
3. Otherwise pick the first available key in this order:
   `ELEVENLABS_API_KEY`, then `DEEPGRAM_API_KEY`.
4. If neither key is present → clear error to stderr, exit 1.

## Audio

Both providers are asked for raw PCM: 16-bit signed little-endian, mono,
24 kHz.

- **Playback:** feed the PCM to `sounddevice` as an `int16` numpy array and
  block until playback finishes. No `ffmpeg` or system players required.
- **File output (`-o`):** wrap the same PCM bytes into a WAV file using the
  stdlib `wave` module (24 kHz, mono, 16-bit). Output is always WAV regardless
  of the file extension given.

## Rate handling

`-r` is words per minute, matching macOS `say` (default 175). It is converted
to a speed factor `rate / 175`:

- **ElevenLabs:** passed as `voice_settings.speed`, clamped to 0.7–1.2.
- **Deepgram:** the TTS API has no speed parameter. If `-r` differs from the
  default, print a warning to stderr that rate is unsupported for Deepgram and
  continue at normal speed.

## Provider details

### ElevenLabs

- Endpoint: `POST https://api.elevenlabs.io/v1/text-to-speech/{voice_id}`
- Header: `xi-api-key: <key>`
- Query: `output_format=pcm_24000`
- Body: `{ "text": ..., "model_id": "eleven_turbo_v2_5",
  "voice_settings": { "speed": <factor> } }`
- Default voice: `21m00Tcm4TlvDq8ikWAM` (Rachel).

### Deepgram

- Endpoint: `POST https://api.deepgram.com/v1/speak`
- Query: `model=<voice>`, `encoding=linear16`, `sample_rate=24000`,
  `container=none`
- Header: `Authorization: Token <key>`
- Body: `{ "text": ... }`
- Default voice/model: `aura-2-thalia-en`.

## Module structure

```
src/say_e11/
  __init__.py
  cli.py              # argparse, text resolution, orchestration
  config.py           # .env loading, key detection, provider choice
  audio.py            # sounddevice playback + WAV writing
  providers/
    __init__.py       # build_provider(name, key) factory
    base.py           # Provider protocol: synthesize(text, voice, rate) -> bytes (PCM)
    elevenlabs.py
    deepgram.py
```

`cli.py` resolves text, calls `config` to pick a provider, calls the provider
to get PCM bytes, then plays and/or writes via `audio.py`.

## Error handling

- Missing API keys → clear stderr message, exit 1.
- HTTP errors from providers → print status and response body snippet to
  stderr, exit 1.
- No text resolved → stderr message, exit 1.
- Empty text → stderr message, exit 1.

## Dependencies

- `httpx` — HTTP calls to TTS endpoints
- `sounddevice` — audio playback
- `numpy` — PCM buffer for `sounddevice`
- `python-dotenv` — `.env` loading

Build: `pyproject.toml` with a `[project.scripts]` entry `say = "say_e11.cli:main"`.

## Testing

- Provider modules tested with a mocked `httpx` transport (`httpx.MockTransport`)
  — no network, no real API keys.
- `config.py` tested with temp `.env` files and patched environment for the
  selection-order rules.
- `cli.py` text-resolution precedence tested with args/stdin/file combinations.
- `audio.py` WAV writing tested by reading the file back with `wave`; playback
  is not unit-tested (hardware dependent).

## Out of scope

- Streaming playback (synthesize fully, then play).
- Non-WAV output formats.
- Listing available voices.
- Provider speed control beyond the simple factor mapping above.
